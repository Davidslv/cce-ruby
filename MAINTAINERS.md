## Responsibilities

Maintainers:

- Triage issues and review pull requests.
- Uphold the quality gates in [`CONTRIBUTING.md`](CONTRIBUTING.md): tests green,
  coverage maintained, spec conformance unchanged, docs updated.
- Guard fidelity to [`SPEC.md`](SPEC.md) and coordination with the sibling Rust
  implementation.
- Cut releases and keep [`CHANGELOG.md`](CHANGELOG.md) accurate.

## Becoming a maintainer

There is no fixed quota. The path is earned trust, demonstrated over time:

1. Contribute several high-quality, test-first pull requests that respect the
   spec and the quality gates.
2. Help with triage, review, and answering questions.
3. Show good judgement about scope and determinism.

When a contributor has a sustained track record, the lead maintainer may invite
them to join. Maintainer status is granted (and, if necessary, revoked) by the
lead maintainer, per [`GOVERNANCE.md`](GOVERNANCE.md).
