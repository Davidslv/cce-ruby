# Support

Thanks for using cce-ruby. Here is how to get help and what to expect.

## Where to get help

- **Questions / discussion / ideas** — open a
  [GitHub Discussion](https://github.com/davidslv/cce-ruby/discussions) if
  Discussions are enabled, otherwise open an issue.
- **Bug reports** — use the bug report form:
  <https://github.com/davidslv/cce-ruby/issues/new/choose>.
- **Feature requests** — use the feature request form at the same link.
- **Security vulnerabilities** — do **not** use public issues; follow
  [`SECURITY.md`](SECURITY.md).
- **Understanding behaviour** — the specification, [`SPEC.md`](SPEC.md), is the
  authoritative reference. Start with [`docs/getting-started.md`](docs/getting-started.md)
  and [`docs/how-to.md`](docs/how-to.md).

## In scope

- Bugs where CCE's behaviour diverges from [`SPEC.md`](SPEC.md).
- Crashes, incorrect output, or non-deterministic results.
- Documentation that is wrong, missing, or misleading.
- Reasonable feature requests that fit the project's goals.

## Out of scope

- Support for source languages beyond those the chunker ships with (currently
  Python and JavaScript grammars).
- General Ruby, Bundler, or environment setup help unrelated to CCE.
- Help operating a third-party Ollama server.
- Bespoke integration or consulting work.

## What to expect

cce-ruby is maintained by **one person** ([David Silva](https://davidslv.uk)) on
a **best-effort, volunteer basis**. There is **no service-level agreement (SLA)**
and no guaranteed response time. Issues and pull requests are triaged when time
allows. Clear, reproducible reports (versions, steps, expected vs. actual) get
resolved fastest. See [`GOVERNANCE.md`](GOVERNANCE.md) for how decisions are
made.
